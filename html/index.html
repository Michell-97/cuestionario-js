¿QUÉ ES UN SVC?
SVC (Sistema de Control de Versiones)

Es un sistema que va a guardar los cambios de un archivo o varios al paso del tiempo y así facilitar la recuperacion de la versión que se desee. Se puede ver quién y cuándo se hizo la última modificación.

¿QUÉ ES GIT?
Software de control de versiones

Guarda las versiones en forma de copias del conjunto de archivos y al realizar un cambio generaa la copia de todo y si no hubo un cambio da un enlace al archivo que ya tenía almacenado.

¿PARA QUÉ SIRVE EL COMANDO GIT?

Para guardar los archivos en ese directorio (git).

¿PARA QUÉ SIRVE EL COMANDO git init?

Crea un nuevo repositorio en Git.

¿PARA QUÉ SIRVE EL COMANDO git add?

Para agregar un archivo o conjunto de archivos al repositorio que se creó.

¿PARA QUÉ SIRVE EL COMANDO git commit?

Es un mensaje breve para describir lo que se está subiendo al repositorio.

¿PARA QUÉ SIRVE EL COMANDO git push?

Para confirmar que se va a subir los archivos al repositorio.

PASOS PARA SUBIR UN PROYECTO A UN REPOSITORIO
EXISTENTE

git remote -v, git add . (todo) ó git add / (nombre del archivo), git commit -m "comentario" y git push origin

NUEVO

git init, git add . (todo) ó git add / (nombre del archivo), git commit -m "comentario", git remote add origin link con la ruta del repositorio nuevo, git branch -M main y git push -u origin

¿POR QUÉ GIT TE PERMITE TRABAJAR COLABORATIVAMENTE EN UN PROYECTO?

Permite clonar un repositorio para que los colaboradores trabajen en él desde su dispositivo.
